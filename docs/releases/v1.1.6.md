# v1.1.6

`2026-04-29`

## 사용자 체감

- **v1.1.5 업데이트 시 발생하던 크래시 해결** — 기존 사용자(v1.1.4 이하) 가 v1.1.5 로 업데이트할 때 DB 마이그레이션 실패로 앱이 매 실행 즉시 크래시되던 문제 수정. v1.1.5 에 머물러 크래시 루프에 빠져있던 사용자도 v1.1.6 받으면 자동 복구됨. Room 의 transaction rollback 안전망 덕분에 데이터 손실 없이 정상 마이그레이션 수행.
- **네이티브 광고 노출 빈도가 페어 카드 2 행마다 1 회로 변경** — 기존엔 페어 6 개마다 광고 1 개였는데, 페어 그리드의 시각적 2 행 단위(약 페어 4 개)로 변경. 날짜 헤더로 발생하는 반쪽 행도 1 행으로 카운트. Album · Home 모두 적용.

## 적용 화면

- 모든 화면 (DB 마이그레이션 fix)
- 홈, 앨범 (광고 빈도)

## 내부 변경

- 마이그레이션 회귀 자동 차단 — `MigrationTest` (instrumented) 신설로 Schema 검증 · 데이터 보존 · FK 보존 3 종 자동화. 이번 사고 같은 schema mismatch 가 PR 단계에서 검출됨.
- GitHub Actions CI 도입 — JVM 단위 테스트 + Roborazzi 시각 회귀 + 마이그레이션 실기 테스트 PR 마다 자동 실행.
- `/cut-release` Skill 도입 — 매 릴리즈 전 위 테스트들을 HARD GATE 로 강제 실행, 통과 시에만 release notes 작성·태그 생성.
- 모듈 구조 정리 — `core/ads` 에 잘못 들어가 있던 도메인 개념(`PairListItem`, `PremiumFeature`) 을 `core/domain` 으로, UI 컴포넌트(`PairShotBannerAd`, `PairShotNativeAdCard`, `RewardedGateDialog`) 를 신규 `core/ads-ui` 모듈로 분리. ArchUnit 룰 위반 14 → 0.
- 미사용 testTag · screenshot baseline 정리, claude-hooks 위치 이동.

## 참고

이전 v1.1.5 는 원본 사라짐 정책 · 합성 안정화 · Roborazzi 시각 회귀 인프라 도입 릴리즈였으나, DB 마이그레이션 SQL 의 결함으로 기존 사용자에게 크래시를 유발했습니다. 본 v1.1.6 은 그 크래시를 fix 하면서 동일 사고 재발 방지를 위한 자동 검증 인프라(MigrationTest · CI · cut-release Skill) 를 함께 도입한 핫픽스입니다.
